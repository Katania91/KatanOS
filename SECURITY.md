# Security Policy

## Supported Versions

Security fixes are provided for the latest published version on the `main` branch.

## Reporting A Vulnerability

Please do not open public issues for security vulnerabilities.

Use GitHub Private Vulnerability Reporting (Security Advisories) for this repository.

When reporting, include:

1. A clear description of the issue and impact.
2. Reproduction steps or a proof of concept.
3. Affected environment (OS, app version, configuration).
4. Any suggested mitigation, if available.

You will receive acknowledgment as soon as possible, followed by validation and remediation steps.
